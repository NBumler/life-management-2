# Internationalization (ngx-translate)

Runtime language switching. Text lives in JSON files, code references keys.

## Setup (once)

```
npm install @ngx-translate/core @ngx-translate/http-loader
```

Provide the loader in `main.ts` (standalone bootstrap) and put translation files
in `src/assets/i18n/`:

```
frontend/src/assets/i18n/
├── en.json
└── hu.json
```

```json
// hu.json
{
  "LOGIN": {
    "TITLE": "Bejelentkezés",
    "SUBMIT": "Belépés",
    "GREETING": "Üdv, {{name}}!"
  },
  "ERROR": {
    "NOT_FOUND": "Nem található"
  }
}
```

## Usage

In templates (preferred):

```html
<h1>{{ 'LOGIN.TITLE' | translate }}</h1>
<p>{{ 'LOGIN.GREETING' | translate:{ name: user.name } }}</p>
```

In code when needed:

```ts
this.translate.get('ERROR.NOT_FOUND').subscribe(msg => ...);
```

## Key naming rules

- Group by feature/screen, UPPER_SNAKE segments: `FEATURE.SECTION.KEY`.
- Mirror the backend error `code`s under an `ERROR.*` namespace so a backend
  `NOT_FOUND` code maps to `ERROR.NOT_FOUND` — one clean translation path.
- Every key must exist in ALL language files. No hardcoded strings in templates.
- Use interpolation params (`{{name}}`) instead of string concatenation.
